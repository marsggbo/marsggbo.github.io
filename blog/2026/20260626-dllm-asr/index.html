2026 | dLLM-ASR 让语音识别推理快 4.44 倍，还不掉精度

原文：dLLM-ASR: A Faster Diffusion LLM-based Framework for Speech Recognition

1. 前言

你有没有想过，我们现在用 LLM 做语音识别（ASR），到底卡在哪儿？

模型精度已经不是问题了——接上个大语言模型，识别率妥妥起来。但推理延迟是个结结实实的大坑：token 一个一个生成，序列越长等得越久，这是自回归（autoregressive）LLM 的先天缺陷。

自然而然就想到了最近很火的 Diffusion LLM（dLLM）。它可以并行生成整个序列，理论上比自回归快很多。但直接把 dLLM 搬来做 ASR，又踩了另一个坑：

  • dLLM 默认从纯随机噪声开始去噪，但 ASR 这个任务其实有很强的先验（你大概知道这段音频会说多少字、说什么）
  • 序列长度不灵活，不同句子长短差很多，固定长度很浪费
  • 去噪步骤固定，有些 token 早就收敛了还在跑，纯属浪费算力

这篇 dLLM-ASR 就是来解这三个矛盾的，最终在 LibriSpeech 上实现了 4.44× 推理加速，识别精度和自回归系统持平。

2. 核心问题：dLLM 用于 ASR 的三大不匹配

先交代下背景。现在主流的 LLM-based ASR 一般是把语音编码器（比如 Whisper）接在 LLM 前面，让 LLM 自回归地输出文字。精度确实上去了，但慢。

dLLM 最有名的代表是 LLaDA，它通过 masked diffusion 并行生成序列。直接把它接上 Whisper 做 ASR（文中称为 Whisper-LLaDA baseline），有几个根本性的矛盾：

矛盾一：从纯噪声开始去噪 dLLM 初始状态是全 mask，没有任何先验。但 ASR 的语音输入其实已经隐含了转录结果的大量信息——长度、音节分布、停顿位置，都是先验。从零开始去噪，等于白白浪费这些信息。

矛盾二：长度不灵活 不同句子长短差异很大，dLLM 需要预设序列长度。短句子填充、长句子截断，都是性能损耗来源。

矛盾三：去噪步骤固定 每个 token 收敛速度不同，但固定步骤下全都要跑完。早就确定的 token 继续参与去噪，纯属浪费。

3. dLLM-ASR 的三招解法

如下图，整个方案可以理解为”在 dLLM 的去噪过程上打三个补丁”：

3.1 ASR Prior 初始化

核心思路：不从纯噪声开始，用语音先验来初始化去噪起点。

具体做法是借用 CTC（Connectionist Temporal Classification）的输出作为初始序列，再加上长度预测。这样一来：

  • 去噪起点已经”不那么随机”了，离最终结果更近
  • 序列长度根据 CTC 预测动态确定，解决了长度不灵活的问题

3.2 Length-Adaptive Pruning

核心思路：动态剪掉冗余 token，让序列长度随去噪过程自适应收缩。

去噪过程中，已经高置信度确定的 token 不再需要继续参与计算。通过置信度阈值裁掉这些 token，每一步的有效序列长度都在减少，计算量自然跟着降下来。

3.3 Confidence-Based Denoising

核心思路：收敛的 token 提前退出去噪循环。

不是所有 token 都需要跑完全部去噪步骤。置信度超过阈值 τ 的 token 直接”冻结”，后续步骤不再更新它。这个 τ 怎么选？如下图是消融实验：

τ 取值对 LibriSpeech clean 和 other 两个测试集的 WER 影响不一样，需要在速度和精度之间找平衡点。

4. 效果怎样

直接看数字：4.44× 推理加速，LibriSpeech 上 WER 与自回归 LLM-ASR 持平。

这个加速比不是靠牺牲精度换来的，三个机制协同发力：

  • ASR Prior 让去噪从更好的起点出发，步骤可以更少
  • Pruning 减少了每步的序列长度
  • Confidence-based 退出减少了每个 token 参与的步骤数

本质上，dLLM-ASR 是在把 dLLM 的”盲目并行去噪”改造成”有先验的自适应去噪”，这也是它能在 ASR 这个强先验任务上大幅提速的核心原因。

5. 一点个人 take

这篇工作的思路挺清晰的——dLLM 做 ASR 的三个不匹配对应三个解法，逻辑链路干净。

但有几个问题我自己会想：

  1. CTC 作为 prior 的鲁棒性：CTC 本身在噪声环境下 WER 就会上去，用它初始化去噪起点，是不是在噪声场景下会把误差传播进去？论文在 LibriSpeech 上验证，这个数据集相对干净，噪声鲁棒性值得进一步验证。

  2. τ 的泛化性：置信度阈值在不同领域数据（方言、口音、专业词汇）上是否还 hold？消融实验只在 LibriSpeech 上做，领域迁移时可能需要重新调。

  3. 与流式 ASR 的兼容性：并行去噪天然不适合流式场景，dLLM-ASR 加速的前提是等整段音频都有了再处理，实时性受限。

当然，作为一篇探索性工作，能在 dLLM 上把这三个问题想清楚并给出 4.44× 的数字，已经是不错的贡献了。后续在更 challenging 的数据集和流式场景上的延伸工作，值得期待。
